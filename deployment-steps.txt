Steps to Push Changes Live to GitHub Pages:

1. Navigate to the Project Directory:
   cd c:\Users\taylo\Desktop\steve-hat-gen

2. Check the Status of the Repository:
   git status

3. Stage All Changes:
   git add .

4. Commit the Changes:
   git commit -m "Update deployment configuration and website content"

5. Push Changes to the `main` Branch:
   git push origin main

6. Monitor the GitHub Actions Workflow:
   - Go to the **Actions** tab in your repository:
     https://github.com/i2gucci/steve-hat-gen/actions
   - Click on the running workflow to monitor its progress and ensure it completes successfully.

7. Verify the Deployment:
   - Go to the **Settings** > **Pages** section of your repository:
     https://github.com/i2gucci/steve-hat-gen/settings/pages
   - Ensure the source is set to the `gh-pages` branch.
   - Visit the live site to confirm the changes are reflected:
     https://i2gucci.github.io/steve-hat-gen/

Debugging Tips:
- If the changes are not reflected:
  1. Check the GitHub Actions logs for errors.
  2. Verify the `gh-pages` branch for the latest files.
  3. Clear your browser cache or use an incognito window.
  4. Force a new deployment with:
     git commit --allow-empty -m "Trigger redeployment"
     git push origin main
